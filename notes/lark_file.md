# Lark File Documentation

The `verilog.lark` file serves as both the grammar source and the living documentation for the project. Metadata tags embedded in comments enable automatic generation of documentation, tests, and coverage reports.

## Metadata Tags

Tags are placed in comments immediately before a grammar rule. Use the format `// TAG: VALUE` with a colon delimiter.

### Core Tags

| Tag | Values | Description |
|-----|--------|-------------|
| `SUPPORT` | `YES`, `NO` (default) | Whether this language construct is supported by the parser |
| `SECTION` | string | Verilog BNF specification section (e.g., "A.1.2 Verilog source text") |
| `BNF` | multi-line | Original BNF from the Verilog spec (continues until next rule) |
| `SV` | multi-line | SystemVerilog extension BNF — parsed identically to `BNF:` |

### Planning Tags

| Tag | Values | Description |
|-----|--------|-------------|
| `PRIORITY` | `HIGH`, `MEDIUM`, `LOW` | Implementation priority for unsupported constructs |
| `DEPS` | comma-separated rules | Auto-generated by `parse_metadata.py --generate-deps`: child rule references extracted from the rule definition. Do not edit manually. |

### Classification Tags

| Tag | Values | Description |
|-----|--------|-------------|
| `SYNTHESIZABLE` | `YES`, `NO`, `PARTIAL` | Whether the construct is synthesizable |
| `EXAMPLE` | string | Inline Verilog snippet demonstrating the construct. Multiple `EXAMPLE:` lines are allowed per rule; all are collected. |

### Example Usage

```lark
// SECTION: A.1.2 Verilog source text
// BNF: module_declaration ::=
//     { attribute_instance } module_keyword module_identifier ...
// SUPPORT: YES
// PRIORITY: HIGH
// SYNTHESIZABLE: YES
// EXAMPLE: module adder #(parameter W=8) (input [W-1:0] a, b, output [W-1:0] sum); assign sum = a + b; endmodule
// DEPS: attribute_instance, module_item, port_declaration
module_declaration: attribute_instance* KW_MODULE MODULE_IDENTIFIER ...
```

# Reference

## BNF vs EBNF Notation

### BNF Symbols

| Symbol | Meaning |
|--------|---------|
| `""` | Encloses a terminal symbol |
| `<>` | Indicates a nonterminal symbol |
| `()` | Indicates a group of valid options |
| `+` | One or more of the previous element |
| `*` | Zero or more of the previous element |
| `?` | Zero or one occurrence |
| `\|` | Separates alternatives |
| `[x-z]` | Letter or digit intervals |
| `[ ]` | Surrounds optional items |
| `{ }` | Surrounds items that repeat zero or more times |

### BNF to EBNF Conversion

**Recursion elimination:**
```
A ::= a A | B
⇒ A ::= a { a } B
```

**Factoring common prefixes:**
```
A ::= a B | a
⇒ A := a [B]
```

## Lark Grammar Rules

- **Terminal**: string, regex, or concat of other terminals (UPPERCASE)
- **Rule**: list of terminals and rules (lowercase)

### Rule Syntax

| Syntax | Meaning |
|--------|---------|
| `"string"` | String literal |
| `/regexp/` | Regular expression literal |
| `(item item ..)` | Group items |
| `[item item ..]` | Maybe (generates `None` if no match when `maybe_placeholders=True`) |
| `item?` | Zero or one instances |
| `item*` | Zero or more instances |
| `item+` | One or more instances |
| `item ~ n` | Exactly n instances |
| `item ~ n..m` | Between n and m instances |
| `\|` | Extends rule definition to next line |

### Tree Shaping Prefixes

| Prefix | Effect |
|--------|--------|
| `_item` | Inlined into containing rule |
| `!item` | Keep all terminals |
| `?item` | Inlined if single child |

See: https://lark-parser.readthedocs.io/en/latest/tree_construction.html

## Regex Cheatsheet

### Special Characters

| Pattern | Meaning |
|---------|---------|
| `\` | Escape special characters |
| `.` | Matches any character |
| `^` | Matches beginning of string |
| `$` | Matches end of string |
| `[5b-d]` | Matches '5', 'b', 'c', or 'd' |
| `[^a-c6]` | Matches any char except 'a', 'b', 'c', or '6' |
| `R\|S` | Matches regex R or S |
| `()` | Capture group / precedence |

### Special Sequences

| Sequence | Meaning |
|----------|---------|
| `\A` | Start of string |
| `\b` | Word boundary |
| `\B` | Not word boundary |
| `\d` | Digit |
| `\D` | Non-digit |
| `\s` | Whitespace `[ \t\n\r\f\v]` |
| `\S` | Non-whitespace |
| `\w` | Alphanumeric `[0-9a-zA-Z_]` |
| `\W` | Non-alphanumeric |
| `\Z` | End of string |
| `\g<id>` | Previously defined group |

### Quantifiers

| Quantifier | Meaning |
|------------|---------|
| `*` | 0 or more (append `?` for non-greedy) |
| `+` | 1 or more (append `?` for non-greedy) |
| `?` | 0 or 1 (append `?` for non-greedy) |
| `{m}` | Exactly m occurrences |
| `{m,n}` | m to n occurrences |
| `{m,n}?` | m to n, as few as possible |

### Assertions

| Pattern | Meaning |
|---------|---------|
| `(?iLmsux)` | Set regex flags |
| `(?:...)` | Non-capturing group |
| `(?P<name>...)` | Named group |
| `(?P=name)` | Match named group |
| `(?#...)` | Comment (ignored) |
| `(?=...)` | Lookahead assertion |
| `(?!...)` | Negative lookahead |
| `(?<=...)` | Lookbehind assertion |
| `(?<!...)` | Negative lookbehind |
| `(?(id)yes\|no)` | Conditional match |
