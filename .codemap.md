# veriforge Codebase Map

A guided tour from package entry to the simulation engines. Each entry links to a
`# cm:ID` anchor embedded in the source. Use `/cm:ID` in your editor or grep to jump.

---

## 1. Package Entry & Public API

- [Root package exports](cm://a1f3b2) — `parse_file`, `parse_files`, `parse_directory`, `Testbench`, `AXI4Master`, etc.
- [Testbench / compile_native re-export](cm://9c4d7e) — convenience re-exports from sim.bench
- [parse_file()](cm://e5b6f8) — parse one Verilog/SV file into a Design; uses .pcache for speed
- [parse_files()](cm://2d1a3c) — merge an explicit file list into one Design
- [parse_directory()](cm://7b9e5f) — scan a directory tree by extension, parse and merge
- [build_testbench()](cm://4c8a1d) — one-call Testbench from a path/Design + optional top name
- [generate_python_testbench_skeleton()](cm://f3d2c6) — emit a Python testbench .py file from a module
- [export_dsl_project()](cm://6e4b9a) — write a Design to Python DSL files

## 4. Semantic Model — Core Types

- [class SourceLocation](cm://3a6e9c) — file/line/column span; attached to every model node
- [class VerilogNode](cm://d4f6b3) — base class for all model objects; provides loc, parent, serialization
- [class Module](cm://b2d4f1) — Verilog module: ports, nets, always blocks, instances, etc.
- [class Design](cm://5c8e7a) — top-level container: list of Module, Package, Interface
- [class Port](cm://9f1b3d) — module port with direction, width expression, type
- [class PortDirection](cm://4e2c6f) — enum: input / output / inout
- [class Expression](cm://7a5d8b) — base for all expression nodes (Identifier, BinaryOp, Literal…)
- [class Identifier](cm://1e3f2c) — name reference in an expression context
- [class BinaryOp](cm://c6b4a9) — binary expression: op, left, right
- [class AlwaysBlock](cm://2f7d5e) — always @(sensitivity) block with body statements
- [class InitialBlock](cm://8c3a1f) — initial block with body statements

## 5. Transforms — Parse Tree → Model

- [tree_to_design()](cm://6d5a2c) — single entry point: Lark Tree + source text → Design

## 6. Analysis Passes

- [class ClockSignal](cm://1a5b6e) — detected clock: name + edge + driven always blocks
- [class ResetSignal](cm://d8c7a3) — detected reset: name + active_low + style (sync/async)
- [extract_clocks_resets()](cm://f2b8e7) — structural clock/reset extraction from always sensitivity lists
- [extract_clocks_resets_hier()](cm://3c9d4f) — hierarchy-aware version; promotes sub-module clocks to ports
- [width inference](cm://4f2e9b) — infer expression bit widths from context

---

## 7. Code Generation & Conversion

- [emit_module()](cm://b7a6d5) — model Module → Verilog text; used for round-trip and DSL export
- [module_to_dsl()](cm://3e2a4c) — model Module → Python DSL source text

---

## 8. Python DSL Builder

- [class Module (DSL)](cm://2b9e4c) — operator-overloaded builder; `with Module("name") as m:` pattern
- [class Signal](cm://7d5f3a) — lazy expression node; operator overloads build model trees

---

## 9. Simulation — Shared Infrastructure

- [class Value](cm://c8a1e6) — 4-state (0/1/x/z) bit-vector; val+mask+width; shared by all engines
- [flatten_module()](cm://9f4b2d) — hierarchy elaboration: flatten submodule instances into flat module
- [class TimedEvent](cm://5a3c7e) — priority queue entry (time, seq, payload) for event scheduling
- [class EventQueueMixin](cm://8b6f1d) — heapq-based event scheduling shared by all 3 engines
- [class CoroutineMixin](cm://f1d5b8) — coroutine lifecycle for timing-control fallback (#delay, @(event))
- [class SignalDictBase](cm://6c4a9f) — base class for signal storage (reference engine uses dict[str, Value])

---

## 10. Reference Engine

- [class SignalHandle](cm://2e7d3b) — testbench-facing read/write proxy to a simulation signal
- [class Simulator](cm://a5c8f4) — top-level reference engine: elaborate, fork, run; engine="reference"
- [class Clock](cm://4d1b6e) — built-in clock generator process
- [class Scheduler](cm://9a7f2c) — IEEE 1364 scheduler: Active → NBA → delta-cycle loop
- [class Process](cm://b3e5d8) — base for all simulation processes (always, initial, testbench)
- [class EvalContext / ExpressionEvaluator](cm://1f4c6a) — 4-state expression evaluation against signal state
- [class StatementExecutor](cm://c2f9a1) — procedural statement execution (blocking/NBA/if/case/for…)

---

## 11. VM Bytecode Engine

- [Opcode enum](cm://5b3d7f) — 74-opcode instruction set for the VM interpreter
- [class Compiler (VM)](cm://8c1e4a) — AST → bytecode compiler; run once at elaboration time
- [class Interpreter / execute()](cm://e3f1b4) — stack-based bytecode interpreter; inner loop
- [class VMScheduler](cm://6d8a2f) — VM engine scheduler; wraps Cython fast interpreter + C delta loop

---

## 12. Compiled Cython Engine

- [generate_pyx()](cm://b1c5e8) — AST → design-specific .pyx source; emits C struct + process fns
- [compile / compile_design()](cm://3d7f4a) — .pyx → .pyd/.so via Cython + C compiler; content-hash cached
- [compile / compile_design() - timeout](cm://3d7f4b) — timeout for compile to finish
- [class CompiledScheduler](cm://f8e1c2) — compiled engine scheduler; drives inlined C delta loop

---

## 13. Testbench Framework — Plan & Runtime

- [class ClockSpec](cm://9d6c3f) — immutable clock descriptor: name + edge + period_hint
- [class ResetSpec](cm://2a8b7e) — immutable reset descriptor: name + polarity + style
- [class ClockDomain](cm://7e9f8c) — one clock domain: ClockSpec + ResetSpec + interface list
- [class InterfaceBinding](cm://c4f5d1) — binds a detected interface to a domain with role + prefix
- [class TestbenchPlan](cm://5b4a9e) — immutable inferred plan: all domains + interface bindings
- [class PlannerOverrides](cm://1f6e5b) — dataclass for user overrides passed to build_plan()
- [build_plan()](cm://4b3d2a) — inference pipeline: clock/reset analysis + interface detection → TestbenchPlan
- [class Domain](cm://e3f1b5) — live clock domain: period, reset, DomainCoordinator, step/reset methods
- [class Testbench](cm://8a7c9d) — main user-facing bench: wraps plan + Simulator + Domain objects
- [make_bench()](cm://6d5f2c) — convenience wrapper: source text or DSL Module → Testbench

---

## 14. Interface Proxies (High-Level)

- [class AXIStreamProxy](cm://f2b9e7) — AXI-Stream proxy: put() / get() / expect(); role=master or slave
- [class AXILiteProxy](cm://3c9d5f) — AXI-Lite proxy: read() / write(); role=master or slave
- [class AXI4Proxy](cm://1a5c6e) — AXI4 proxy: read() / write() with burst support
- [class MemBusProxy](cm://d8c8a3) — MemBus proxy: read() / write() for SRAM/BRAM-style ports
- [class StreamProxy](cm://4f3e9b) — simple ready/valid Stream proxy (Pulp-style)

---

## 15. Engine-Native Lowering

- [compile_native()](cm://b7b6d5) — lower a Testbench + lowerings map → LoweredDesign for VM/compiled
- [class LoweredDesign](cm://3e3a4c) — synthesized wrapper module: DUT + bench fragments; .run() batches
- [class AXIStreamSourceLowering](cm://2b9f4c) — DSL FSM: drive fixed beat sequence with PRNG backpressure
- [class AXILiteSlaveLowering](cm://7d6f3a) — DSL FSM: memory-backed AXI-Lite slave responder
- [class AXI4SlaveLowering](cm://c9a1e6) — DSL FSM: memory-backed AXI4 slave with INCR burst + WSTRB

---

## 16. Low-Level Endpoints

- [class DetectedInterface](cm://5a4c7e) — protocol + prefix + role + port list from detection
- [detect_interfaces()](cm://9f5b2d) — port-name heuristics → list of DetectedInterface
- [class AXIStreamSource](cm://8b7f1d) — tick_pre/sample_pre/tick_post endpoint: drives tvalid/tdata/tlast
- [class AXIStreamSink](cm://f2d5b9) — captures incoming AXI-Stream beats; validates protocol
- [class AXILiteMaster](cm://6c5a9f) — drives AW/W/B/AR/R channels; blocking transaction API
- [class AXI4Master](cm://2e8d3b) — drives AXI4 channels; INCR burst read/write
- [class MemBusMaster](cm://a6c9f4) — synchronous SRAM/BRAM-style master; .write() / .read()
- [class AXILiteResponder](cm://1f5c6a) — responds to DUT AXI-Lite master; .memory dict; auto-ticks
- [class AXI4Responder](cm://7e9b5d) — responds to DUT AXI4 master; .memory dict; auto-ticks
- [class MemBusResponder](cm://c3f9a2) — responds to DUT MemBus master; .memory dict; auto-ticks
- [class PauseGenerator](cm://b4e6d8) — callable pause policy: random backpressure for source/sink endpoints
- [class DomainCoordinator](cm://9a8f2c) — per-domain endpoint collection; drives tick_pre/sample_pre/tick_post
- [class MultiDomainRunner](cm://4d2b6e) — cross-domain step(); handles truly-async multi-clock testbenches

## Architecture Summary

```
verilog source text
      │
      ▼ preprocessor.py        # cm:8f2d5b
      │
      ▼ verilog_parser.py      # cm:1c7a4e  Lark Earley → CST
      │
      ▼ transforms/            # cm:6d5a2c  CST → semantic model
      │
      ├─▶ analysis/            # cm:f2b8e7  clock/reset, widths, lint
      ├─▶ codegen/             # cm:b7a6d5  model → Verilog text
      ├─▶ dsl/                 # cm:2b9e4c  Python DSL builder
      ├─▶ sim/
      │   ├─ reference engine  # cm:9a7f2c  pure Python scheduler
      │   ├─ vm engine         # cm:6d8a2f  bytecode + Cython C loop
      │   ├─ compiled engine   # cm:f8e1c2  design-specific Cython
      │   └─ bench/            # cm:8a7c9d  Testbench + proxies
      ├─▶ refactor/            # cm:2a9b7e  hierarchy tools
      └─▶ cli/                 # cm:7f9a8c  veriforge commands
```
